# 修正内容の確認: 横向きレイアウト・セーフエリアの改修

横向き（ランドスケープ）時の操作性改善およびノッチ対応の強化を行いました。

## 1. セーフエリア (ノッチ) 対応の強化
iPhoneを横持ちにした際、左右のノッチ（パンチホール）にコンテンツが隠れてしまう問題に対応しました。
- `src/index.css` にて、セーフエリア取得のCSSクラス（`pl-safe`, `pr-safe`, `px-safe` 等）を新設しました。
- 各安全マージンには、`max(env(safe-area-inset-*), 16px)` のようにフォールバック値を持たせ、CapacitorなどWebView実装の都合で0が返却される環境でも一定の余白を担保しました。
- メインのコンテナに `px-safe` を適用したことで、画面全体がノッチ領域を自動的に避けて表示されるようになります。

## 2. モバイル端末の横向き（Landscape）レイアウトの改善
モバイル端末を横向きにした際、画面全体がビューアーで埋まってしまいパラメーターパネルを操作できなくなる問題を解決しました。
- Tailwind CSSの `landscape:` と `portrait:` プレフィックスを使用し、縦持ち・横持ちによってUIを出し分けるようにしました。
- **縦持ち時 (Portrait)**: 従来通り、上部にビューアー、下部にパラメーターのアコーディオンを配置します。
- **横持ち時 (Landscape)**: 画面を左右に分割するPCのようなレイアウトに切り替わります（グリッド比率 1 : 2）。
    - 画面左側（1/3）にパラメーターパネルの全体を表示し、上下にスクロールできるようにしました。
    - 画面右側（2/3）に3Dビュー/TOPマップと被写界深度(DoF)の結果を表示し、画面の高さいっぱい(`100dvh`)を効率よく使用します。
- ParamaterPanel コンポーネントは内部で `h-fit` 制限を持っていたため、これを解除し、柔軟にスクロールさせられるよう改修しました。

これらの変更によって、Web環境だけでなくネイティブ化（Capacitor経由）されたiOSアプリとして動かした場合でも、画面の向きに依存しない快適な操作感を提供できる設計になりました。
