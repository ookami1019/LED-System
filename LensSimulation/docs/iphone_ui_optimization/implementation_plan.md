# 実装計画: iPhone向けUI最適化

iPhoneなどのノッチ付きデバイスでの表示崩れを修正し、スライダーの操作性を向上させます。

## 1. ノッチ（セーフエリア）対応

### 1-1. Viewportの設定変更
`index.html` の `viewport` メタタグに `viewport-fit=cover` を追加します。これにより、コンテンツが画面全体（ノッチの裏側まで）に広がるようになります。

### 1-2. パディングの調整
`App.tsx` のルート要素またはヘッダー部分に、CSSの `env(safe-area-inset-top)` を使用したパディングを追加します。これにより、タイトルがノッチに隠れないように自動的に位置が下がります。

## 2. スライダー操作性の向上

### 2-1. スライダースタイルの刷新
`index.css` の `input[type=range]` に対するスタイルを調整します。
- **つまみ（Thumb）**: 現在の 18px から 28px 程度に拡大し、指で触れやすくします。
- **トラック（Track）**: 現在の 4px から 6px 程度に太くし、視認性と操作性を高めます。
- **配置調整**: つまみのサイズ変更に合わせて、トラック内での垂直位置を調整します。

## 変更対象ファイル
- `index.html`: `viewport-fit=cover` の追記
- `src/index.css`: スライダーのサイズ設定変更
- `src/App.tsx`: ヘッダー部分へのセーフエリア余白追加

## 期待される結果
- iPhoneで閲覧した際、タイトルがノッチに被らず正しく表示される。
- 焦点距離や絞りなどのスライダーが指で操作しやすくなる。
